# 门店业务调研与需求洞察

## 1. 调研范围

业务调研材料覆盖华南区直营与加盟女装门店的一线运营场景，涉及新人带教、商品讲解、活动执行、售后处理、知识传递和店长审批。单店通常同时管理数百个SKU，导购需要在顾客等待时间内快速确认商品和规则信息。

## 2. 新人知识需求

新人入职前30天的高频问题集中在四类：商品基础、活动规则、售后边界和操作流程。商品基础问题约占六成，其中尺码版型、面料洗涤和相似款差异最容易造成不确定。新人并非只是不熟悉商品，而是担心错误承诺、拿错尺码和售后责任，因此倾向于重复询问店长。

产品要求：
- 支持口语化、简称、方言和常见错别字；
- 尺码、面料和护理回答必须保留条件；
- 每个商品知识卡同时展示能说什么与不能说什么。

## 3. 店长答疑与审批负担

店长日常被频繁询问库存调货、售后边界、活动细节、折扣权限和系统操作。常规知识和权责审批混合在一起，造成旺场时多人同时找店长、信息口头传递和处理结果无法追溯。

产品要求：
- 常规商品和规则问题由导购自助查询；
- 折扣、赠品、退换、赔偿、改价和投诉自动生成店长确认任务；
- 审批结果记录问题、参考依据、最终回复和处理说明。

## 4. 客诉与合规风险

高风险主要来自：
1. “绝对不起球、完全不缩水、所有人都显瘦”等绝对化承诺；
2. 特价、会员、满减、价保和退换规则被错误解释；
3. 羊毛、羽绒、缎面等商品护理建议错误；
4. 跨店、补差、免费服务和特殊通融被超范围承诺。

产品要求：
- 识别“就说、肯定、绝对、别让店长知道”等诱导表达；
- 禁语与风险边界使用醒目样式；
- 复杂规则只引用当前生效版本；
- 质量边界和经济损失问题必须人工处理。

## 5. 知识分散与版本损耗

商品信息可能分别存在吊牌、纸质卖点卡、PDF、微信群、收银台规则和员工手册中；活动信息还会经过总部、区域、店长、早班和晚班多级转述。信息在传递中被简化、遗漏或变形，兼职和中途入职人员尤其容易获得错误口径。

产品要求：
- 商品、活动和售后形成唯一知识入口；
- 文档必须包含版本、生效时间、失效时间、适用范围和风险等级；
- 过期文档不参与回答；
- 新旧规则冲突时优先当前生效且版本更高的文件。

## 6. 接待现场的交互约束

顾客当面提问时，导购需要快速查看答案。有效信息应控制在手机一屏内，并采用四层结构：
1. 一句话核心结论；
2. 可直接复述的顾客话术；
3. 禁语与边界提醒；
4. 应对或升级动作。

产品要求：
- 优先展示短结论和话术；
- 详细来源折叠显示；
- 商品编码、别名和错别字均可检索；
- 扫码、商品编码、语音和文字输入均作为正式查询入口。

## 7. 顾客匹配与连带销售

商品推荐需要综合人群、身形、场景和风格，而不是简单输出热门款。连带推荐遵循四条逻辑：基础品类互补、场景统一、季节递进和身形适配。

产品要求：
- 顾客标签可多选；
- 推荐结果说明命中标签；
- 主品关联3件商品并标注推荐逻辑；
- 话术不得强迫消费或对身材作负面评价。

## 8. 实时数据边界

库存、货位、价格、调货时间、排班、薪资和设备状态属于实时业务数据。静态知识库无法可靠回答，必须接入POS、ERP、WMS、HR或门店运营系统。

产品要求：
- 检测实时数据问题并拒绝猜测；
- 明确提示应查询的系统或负责人；
- 不将模型常识作为企业实时数据来源。

完整结构化结论见 `data/research_findings.csv`，高频问题见 `data/question_bank.csv`。
