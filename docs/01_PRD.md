# 门店智伴产品需求文档

## 1. 产品目标

门店智伴用于缩短导购查询时间、减少错误承诺、降低店长重复答疑、统一门店执行口径，并通过审批、反馈、知识迭代和运营分析形成多门店管理闭环。

## 2. 用户角色

### 导购

- 扫码、编码、语音和文字查询
- 查看本店与附近门店库存
- 获取商品卖点、尺码、搭配和顾客话术
- 查看自己的问答、审批结果和通知
- 提交回答反馈和未建档商品申请

### 店长

- 查看本门店全部问答
- 处理本门店常规高风险问题
- 修改最终话术、填写处理备注、转线下或升级运营
- 管理本门店Bad Case，提交知识库优化申请
- 补充未建档商品资料
- 查看门店经营辅助看板

### 区域运营

- 管理区域内全部门店工单和反馈
- 终审店长升级工单
- 发布规则、维护商品卡和推荐关系
- 聚合同类Bad Case并分配责任人
- 审核新品建档
- 查看区域对比、知识健康度和操作日志

## 3. 导购问答

### 查询入口

- 吊牌条码或二维码扫描
- 商品编码手动输入
- 商品名称、别名、拼音和首字母搜索
- 语音和文字自然语言提问
- 常见问题快捷按钮

### 回答结构

1. 顾客沟通话术
2. 高风险或人工确认提示
3. 禁语与风险边界
4. 核心结论
5. 适用条件和下一步动作
6. 依据、版本和相关度

### 旺场简洁模式

仅展示顾客话术、风险提示和禁语，其他信息折叠。

## 4. 库存与调货

- 商品卡展示本门店各尺码库存
- 库存状态分为充足、紧张、断货
- 附近门店展示可调尺码、数量和预计到店时间
- 连带推荐优先展示本店有货商品
- 库存未锁定前禁止承诺一定有货

## 5. 高风险工单

### 风险提醒

- 侧边栏区分高风险和中风险数量
- 超过24小时未处理的工单置顶并标红
- 默认按超时、审批层级、风险等级和提交时间排序

### 工单详情

- 导购、门店、提交时间和状态
- AI参考结论、话术、风险和规则依据
- 审批回复和处理备注
- 确认回复、修改后回复、转线下处理
- 店长可升级区域运营
- 审批结果同步到导购通知和问答记录

### 批量处理

- 全选当前页、全选高风险
- 预置备注模板
- 批量确认、转线下或分配门店店长
- 执行前二次确认

## 6. Bad Case闭环

### 状态

- 待处理：尚未分配责任人
- 处理中：已分配并正在分析
- 已优化：知识、话术或系统已修改并完成复测
- 已关闭：问题不成立或无需处理

### 类型

知识库缺失、答案错误、边界模糊、话术不准、来源不匹配、规则过期、诱导违规、系统错误。

### 处理信息

- 原问题、AI回答、导购反馈、门店和时间
- 责任人、优化动作、关联规则/商品、版本
- 效果验证和同类型问题再次出现数量
- 店长可提交区域知识优化申请
- 区域运营可分配给运营或门店负责人

## 7. 数据看板

### 核心指标

- 累计提问、人均查询
- 导购活跃率、平均使用时长
- 高风险问题占比和趋势
- 零结果率
- 答案有帮助率
- Bad Case解决率
- 客诉记录数（接入后）

### 分析视图

- 高频问题和零结果问题Top10，附问题类型
- 门店意图分布对比
- 按门店和导购拆分的高风险问题
- 门店使用排行和状态
- 门店与区域平均对比
- 规则、商品和Bad Case运营动作效果追踪

### 导出

支持Excel和PNG图片导出。

## 8. 新品建档

### 未识别扫码

系统显示未建档状态，不回答商品卖点、库存或价格。

### 导购提交

填写吊牌名称、类别、现场备注和照片。

### 店长补充

补充编码、面料、版型、适用顾客、标签、卖点、搭配、尺码、异议、话术和禁语。

### 区域运营发布

审核价格、版本、生效时间和失效时间；发布后：

- 写入商品知识库
- 建立条码映射
- 为各门店和尺码建立零库存记录
- 记录操作日志

## 9. 规则和商品管理

### 规则

区域运营可发布新版本，配置类型、有效期、风险、关键词和正文。导购与店长只读。所有用户首次进入时需确认已读。

### 商品

区域运营可编辑单个商品，也可批量更新版本、有效期、尺码提示、禁语和标签。

### 连带推荐

区域运营可维护固定推荐关系；系统同时根据标签和库存补充动态推荐。

## 10. 权限与安全

- 导购仅访问自己的记录
- 店长仅访问本门店数据
- 区域运营访问区域内全部门店
- 前端页面与FastAPI接口双重校验
- 所有审批、升级、规则发布、商品更新、Bad Case处理和新品发布写入审计日志
- 角色或门店切换时清空会话
- 同一回答只能提交一次反馈
- 全局异常捕获，用户端展示友好提示，后台记录错误日志

# 核心交互与运营闭环

## 导购端交互原则

1. 旺场查询必须在3秒内进入可读结果。
2. 结果第一屏只呈现顾客话术、禁语和库存状态。
3. 商品编码、扫码、文字、语音和最近查询共用统一查询链路。
4. 顾客标签按身形、场景、风格和人群组织，推荐结果优先保证本店有货。
5. 商品、连带推荐和问答结果均可进入同一算价购物车。

## 工单SLA

| 工单类型 | 处理角色 | 加急阈值 | 超时阈值 |
|---|---|---:|---:|
| 门店常规高风险 | 店长 | 2小时 | 24小时 |
| 店长升级问题 | 区域运营 | 即时置顶 | 12小时 |
| 中风险口径确认 | 店长 | 4小时 | 24小时 |

## Bad Case状态

| 状态 | 定义 | 可执行动作 |
|---|---|---|
| 待处理 | 未分配责任人 | 分类、分配、设置期限 |
| 处理中 | 已进入处理 | 补充证据、关联规则/商品、记录优化动作 |
| 已优化 | 知识或话术已修改并复测 | 观察同类问题是否再次出现 |
| 已关闭 | 问题不成立或无需处理 | 保留完整处理记录 |

## 数据链路

```text
导购查询 → 问题与风险日志 → 高风险工单/Bad Case
商品卡/连带推荐 → 快捷算价购物车 → 使用事件
规则发布 → 已读确认 → 规则咨询变化
淡场练习 → 个人正确率 → 门店成长曲线
运营动作 → 操作日志 → 前后7天问题变化
```
