# 产品方案总览

## 产品定位

门店智伴服务于多门店服装零售组织，把导购即时查询、店长风险审批和区域运营知识治理连接为同一条业务链路。

## 产品价值

- 导购：减少打字、翻资料和反复询问，快速获得可直接复述的话术、库存和规则依据；
- 店长：把常规知识查询与需担责的审批分离，优先处理超时和高风险问题；
- 区域运营：统一商品与规则口径，聚合同类问题，追踪门店差异和运营动作效果；
- 企业：保留审批、知识更新和商品发布的操作记录，降低权责不清和越权承诺风险。

## 数据规模

- 15款结构化商品知识卡；
- 15条商品条码映射；
- 183条分门店、分尺码库存记录；
- 45条固定连带推荐关系；
- 商品标签动态推荐；
- 3套门店制度及章节化知识；
- 289条高频与困难问题库；
- 85条固定AI评测用例；
- 多门店、多角色演示业务数据。

## 核心业务链路

### 导购查询

扫码/编码/语音/文字 → 商品或规则检索 → 库存与调货 → 话术和禁语 → 反馈。

### 风险审批

高风险问题 → 店长一级审批 → 超权限问题升级区域运营 → 最终话术同步导购。

### 知识治理

回答反馈 → Bad Case分类 → 责任人 → 关联商品/规则 → 版本更新 → 效果验证。

### 新品建档

未知条码 → 导购提交 → 店长补全 → 区域运营审核 → 商品卡、条码和库存档案发布。

### 运营分析

提问、会话、风险、零结果、反馈、门店和操作日志 → 门店看板/区域看板 → Excel与图片导出。

## 设计原则

1. 一线优先：操作入口适合手机、旺场和双手占用场景；
2. 话术优先：首屏先展示能对顾客说的话；
3. 数据有源：商品、库存和规则均展示来源、版本或更新时间；
4. 风险分级：普通问题自动回答，高风险问题人工审批；
5. 权限隔离：导购、店长和区域运营的数据范围不同；
6. 反馈闭环：问题必须有状态、责任人、优化动作和验证；
7. 新品不猜测：未知条码不自动生成商品事实；
8. 操作可追溯：审批、发布和修改保留审计记录。

## 产品价值指标

- 使用价值：导购活跃率、人均查询、平均使用时长。
- 知识价值：知识库覆盖率、零结果率、答案有帮助率。
- 风控价值：高风险拦截率、工单超时率、违规诱导问题数量。
- 管理价值：Bad Case解决率、规则已读率、门店问题闭环时长。
- 销售辅助价值：连带推荐加入算价次数、快捷算价使用次数。
- 培训价值：淡场学习参与量、正确率与导购成长变化。
