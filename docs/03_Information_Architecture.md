# 信息架构与权限

## 导购端

- 导购问答
  - 扫码与编码查询
  - 语音和文字问答
  - 话术、库存、依据和反馈
- 通知中心
- 问答记录（仅本人）
- 顾客需求匹配
- 商品知识卡（只读）
- 连带推荐
- 快捷算价
- 规则中心（只读）

## 店长端

包含导购端全部功能，并增加：

- 问答记录（本门店）
- 店长确认
  - 风险提醒
  - 超时工单
  - 单笔审批
  - 批量处理
  - 升级区域运营
- Bad Case
  - 本门店反馈
  - 责任人和状态
  - 知识优化申请
- 新品建档
  - 补充资料
  - 提交区域审核
- 门店数据看板

## 区域运营端

包含公共业务功能，并增加：

- 问答记录（全部门店）
- 区域工单中心
  - 全部门店
  - 终审升级工单
  - 批量处理和分配店长
- Bad Case和知识优化申请
- 新品建档审核发布
- 商品与推荐配置
- 规则发布和版本管理
- 区域数据看板
- 操作日志

## 权限矩阵

| 模块 | 导购 | 店长 | 区域运营 |
|---|---|---|---|
| 导购问答 | 使用 | 使用 | 使用 |
| 问答记录 | 仅本人 | 本门店 | 区域全部 |
| 商品知识卡 | 只读 | 只读 | 编辑和批量维护 |
| 连带推荐 | 使用 | 使用 | 使用并配置 |
| 规则中心 | 当前规则只读 | 当前规则只读 | 发布、版本和差异对比 |
| 店长确认 | 不可见 | 本门店一级审批 | 区域查看和二级审批 |
| Bad Case | 不可见 | 本门店处理和申请优化 | 区域分配、优化和验证 |
| 新品建档 | 提交 | 补充并送审 | 终审和发布 |
| 数据看板 | 不可见 | 本门店和区域均值 | 区域汇总和门店对比 |
| 操作日志 | 不可见 | 不可见 | 查看 |

## 审批分级

```text
高风险问题
  ↓
店长一级审批
  ├─ 常规门店权限：确认、修改或转线下
  └─ 超出门店权限：升级区域运营
          ↓
      区域运营终审
```

## 数据隔离

- 用户账号绑定门店和区域
- 店长查询自动附加本门店条件
- 区域运营可选择区域内门店或全区域
- API通过`X-User-Id`解析角色，再校验数据范围
- 操作日志保存操作人、角色、门店、实体、修改前后内容和时间

## 会话与数据隔离

- 切换账号或门店时清空聊天上下文、扫描结果和临时会话。
- 店长查询条件强制限定所属门店。
- 区域运营可以切换门店或选择全部门店。
- 页面隐藏不作为唯一权限手段，API层再次验证角色和数据范围。
