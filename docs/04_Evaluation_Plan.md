# AI知识助手评测方案

## 1. 数据集

- `question_bank.csv`：覆盖商品、活动会员、退换投诉和困难输入的完整问题库；
- `eval_cases.csv`：固定评测集；
- `inventory.csv`：库存、价格和调货查询测试数据；
- 困难类型：错别字、简称、拼音首字母、多意图、版本冲突、过期规则、无依据、诱导违规和实时库存。

## 2. 自动评测指标

### 意图准确率
评估商品知识、搭配话术、活动会员、售后规则、投诉处理和其他分类。

### 风险准确率与高风险召回
检查退换、赔偿、活动叠加、客诉、越权优惠和绝对化承诺是否进入高风险流程。

### 来源命中@3
预期商品或规则是否出现在前三条检索来源中。

### 拒答准确率
排班、薪资、设备和知识范围外问题是否停止猜测；已接入的库存和价格问题不应被误拒答。

### 库存查询正确性
检查商品编码、尺码、本店库存、附近库存、库存状态和调货时效是否与库存数据源一致。

### 话术可用性
人工检查话术是否可直接复述、禁语是否明确、下一步动作是否可执行，旺场模式是否在首屏显示必要信息。

### 管理闭环可用性
检查高/中风险数量、超时置顶、店长与区域运营数据范围、一级/二级审批、批量处理、导购通知同步、Bad Case状态流转、知识优化申请、新品建档、审计日志和看板导出。

## 5. Bad Case分类

| 类型 | 典型原因 | 处理方式 |
|---|---|---|
| 商品错召回 | 别名、错别字、相似款 | 编码精确匹配、别名加权、拼音首字母 |
| 版本错误 | 旧规则与新规则冲突 | 有效期过滤、历史版本隔离、版本对比 |
| 风险漏判 | 越权表达未进入店长确认 | 风险词典、诱导指令识别、低置信度转人工 |
| 库存错误 | 门店、尺码或更新时间不一致 | 库存服务校验、更新时间展示、调货降级 |
| 过度拒答 | 已有数据仍拒绝回答 | 区分已接入实时数据与未接入数据 |
| 话术不可用 | 太长、术语过多 | 话术置顶、旺场简洁模式、长度约束 |
| 重复反馈 | 连续点击产生垃圾数据 | query_id唯一反馈约束 |
| 权限越界 | 导购访问管理页面 | 页面级角色校验与会话隔离 |


## 6. 权限与管理测试

自动化测试覆盖：

- 导购无法访问管理接口；
- 店长仅能查看本门店工单和反馈；
- 区域运营可查看区域全部门店；
- 店长升级后工单从门店队列转入区域队列；
- 审批结果生成导购通知；
- 未识别条码进入新品建档；
- 条码映射能够解析到商品编码；
- 看板返回活跃率、使用时长、零结果率和Bad Case解决率。
